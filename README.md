# UX AI Governance Templates

Templates, checklists and practical tools for making AI-supported workflows usable, reviewable and governable.

This repository focuses on the operational side of AI governance: not abstract principles, but reusable structures that help teams work with AI safely and effectively.

## Core idea

AI governance only works when people can actually use it.

Policies are important, but teams also need practical templates, review questions, escalation paths, quality checks and documentation habits that fit into real workflows.

## Focus areas

- human-in-the-loop review
- AI-supported content workflows
- prompt review
- image generation review
- UX and brand quality checks
- decision documentation
- risk escalation
- governance for regulated communication environments

## What this repository may contain

- review checklists
- prompt templates
- decision logs
- approval workflows
- quality gates
- risk matrices
- escalation templates
- measurement templates
- documentation examples

## Example use cases

- reviewing AI-generated content
- checking AI-generated images before publication
- documenting human decisions in AI-supported workflows
- deciding when AI output needs escalation
- evaluating whether a prompt or workflow is ready for reuse
- translating governance principles into everyday work

## Working principles

- Human judgment remains explicit.
- Risk is documented before scale.
- Quality gates are part of the workflow.
- Templates should reduce uncertainty, not create bureaucracy.
- Governance should make good work easier.
